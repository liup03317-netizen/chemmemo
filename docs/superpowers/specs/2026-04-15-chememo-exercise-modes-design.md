# ChemMemo 改名与多题型训练设计稿（中学化学方程式）

## 1. 背景与目标
- 产品英文名确定为：ChemMemo（避免与既有品牌相似导致侵权风险）
- 核心目标：在“选择题”之外提供更贴近中高考评分点的多样训练方式，提升记忆效率与迁移能力（会写、会判错因、会配平、会补全、会默写）
- 关键约束：默写模式不要求键盘输入下标、反应条件、气体/沉淀符号；通过结构化交互完成考点覆盖

## 2. 命名改造范围（ChemLingo → ChemMemo）
### 2.1 需要替换的内容
- UI 文案：Header 品牌名、可能出现的页面标题/介绍文案
- LocalStorage 存档 key：从 chemlingo-storage → chemmemo-storage

### 2.2 存档迁移策略
- 启动时检测：
  - 若 chemmemo-storage 不存在且 chemlingo-storage 存在，则拷贝旧数据到新 key 并继续使用新 key
  - 若两者同时存在，以新 key 为准（避免覆盖）

## 3. 题型系统升级（非选择题）
### 3.1 总体原则
- 题面展示与判题不依赖“把条件写进纯文本”
- 方程式展示统一走结构化渲染（条件在箭头上方；无条件不显示；↑/↓ 作为方程式正确性要素）
- 判题输出统一返回：
  - 是否正确
  - 标准答案（用于展示）
  - 错因列表（用于解释“错在哪里”）

### 3.2 题型列表（优先级顺序）
1) 配平输入（优先实现）
2) 方程式填空（词块/拖拽/点选）
3) 匹配/连线（高频刷题）
4) 结构化默写（不键盘输入下标/条件/符号）

## 4. 题型设计细则
### 4.1 配平输入（Balance）
**题面**
- 给出骨架方程式（物质完整，系数为空/默认 1）
- 条件以箭头上方展示（无条件隐藏）

**作答交互**
- 每个物质前提供系数控件：
  - 步进器（+ / -）
  - 或数字键盘输入（仅数字，不涉及下标、符号）
- 默认 1 的显示策略：可配置为“显示 1”或“空代表 1”

**判题与错因**
- 元素守恒检查：定位到具体元素（例如 O 不守恒：左 2 右 1）
- 最简整数比检查：提示“系数可约分”
- 漏写/未填系数检查（按考试模式严格或宽松）

### 4.2 方程式填空（Cloze）
**题面**
- 反应物/生成物/条件可按模板留空
- ↑/↓ 可作为单独空位或以按钮形式附着在生成物上

**作答交互**
- 物质采用词块点选/拖拽填空（避免打字与下标输入）
- 条件采用标签点选（点燃/加热/催化剂/高温高压等）
- ↑/↓ 采用按钮切换

**判题与错因**
- 漏填项：缺少生成物/反应物/条件/符号
- 错填项：物质不匹配、条件错误、符号错误

### 4.3 匹配/连线（Match）
**题面形式**
- 条件 ↔ 方程式
- 现象/用途 ↔ 方程式
- 反应类型 ↔ 方程式

**作答交互**
- 左右列表点击配对，或拖拽连线

**判题与错因**
- 指出错配原因（例如：正确方程式生成气体↑，你匹配到沉淀↓现象）

### 4.4 结构化默写（Recall）
**题面**
- 给出反应物（可选给“有无条件提示/条件空位”）

**作答交互（不键盘输入）**
- 生成物：从物质词库中选择并排序
- 系数：步进器填写
- 条件：标签点选
- ↑/↓：按钮切换

**判题与错因**
- 与方程式标准答案逐项比对（物质、系数、条件、符号）
- 输出“错因”提示：缺条件/条件错/缺符号/配平不对/生成物不全等

## 5. 技术改造点（面向现有代码）
### 5.1 数据模型
- Question 扩展为支持结构化方程式视图（equationView）
- 后续每种题型建议使用更细的 payload（例如 balancePayload、clozePayload），以避免 prompt 文本承载逻辑

### 5.2 UI 组件
- 方程式统一渲染组件（已存在 EquationLine，需保持“无条件隐藏”行为）
- 题型渲染器：根据 Question.type 渲染不同交互
- 判题结果面板：展示标准答案 + 错因列表 + 解析

## 6. 验收标准
- 品牌名全局显示为 ChemMemo，旧存档不丢失
- 默写/填空不要求键盘输入下标、条件、↑/↓，但能正确考察这些要点
- 每次答错可读到明确错因（至少覆盖：条件缺失/错误、↑/↓缺失/错误、生成物错误、配平不守恒/非最简）
